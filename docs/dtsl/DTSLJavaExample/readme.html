This is an example of how to use DTSL in 'stand-alone' mode (i.e. outside of the Eclipse environment). DTSL is mainly written in Java and so can be used by either a Java program, as in this example, or by a Jython (Python for Java) program.
Note
For information on locating the example files and extracting them for use either on the command-line or in Arm Development Studio, see the installation instructions.

Purpose and scope

The Debug and Trace Services Layer (DTSL) is a software layer that sits between the Debugger and the RDDI target access API. The Debugger uses DTSL to create target connections, configure the target platform to be ready for debug operations, and communicate with the debug components on the target.
As a power user of the Debugger, you might need to use DTSL as part of new platform support, to extend the capabilities of the Debugger, to add support for custom debug components, or to create your own Java or Jython programs which interact with your target. These examples illustrate these uses.

Getting Started with the DTSL Examples

The DTSL examples are provided in [install dir]\examples\DTSL_examples.zip. To access these, either import the examples into a workspace, or unzip them into a writable folder.
Some additional files are provided in the [install dir]\sw\DTSL folder. To use the examples you must either import the DTSL library DTSL.zip from this folder into the workspace, or unzip it into a writable folder. This folder also contains the DTSL Java docs converted into a PDF document DTSLJavaDocs.pdf and the zipped up content of the DTSL Javadocs com.arm.debug.dtsl.docs.zip.
If you use the Development Studio Examples Import Wizard to import the DTSL examples, both the DTSL examples [install dir]\examples\DTSL_examples.zip and the DTSL library [install dir]\sw\DTSL\DTSL.zip will be imported automatically for you into your workspace.
To import the DTSL examples using the Development Studio Examples Import Wizard:
  1. Select File → Import... to open the Import Selection dialog.
  2. Expand the Arm Development Studio group and select Examples and Programming Libraries.
  3. Click Next to open the Import Development Studio Examples and Programming Libraries dialog.
  4. Expand the Examples group to view the example categories.
  5. Select Debug and Trace Services Layer (DTSL). Notice that the DTSL library is also selected for you automatically from the Programming Libraries.
  6. Click Finish to import the selected examples into your workspace.
See the readme.txt text files within each sub-directory for further information.
