Jython Progress Monitor Example - Arm®Development Studio

This example illustrates the use of a Jython script in the Debugger with the Progress Monitor to keep the user updated.
Note
For information on locating the example files and extracting them for use either on the command-line or in Arm Development Studio, see the installation instructions.

Purpose and scope

This example illustrates the use of a Jython script in the Debugger with the Progress Monitor to keep the user updated on the progress of execution of the example. This example prints some sentences (deliberately slowly) each containing a number of words, and the Progress Monitor shows the progress of the printing of the words and sentences. This example is intended to illustrate features of the Debugger - it is not intended to be a tutorial on Jython.
This example is configured to use the FVP model of Versatile Express with Cortex-A9x1 Core Tile (Cortex-A9x1 FVP model), and no extra hardware or software is required. No instructions are actually executed on the model - the model connection is just a convenience for the debugger. A ready-made launch configuration jython_progress-FVP.launch is provided for this.

Running the example

  1. Select Run → Debug Configurations....
  2. Select the jython_progress-FVP from the list of Generic Arm C/C++ Application configurations.
After the Cortex-A9x1 FVP model has started up, and the Debugger connected to it, the following command is executed:
  1. source jython_progress/progress_monitor.py
    This command runs the Jython script.
The full output in the Command view looks like this:
Connected to stopped target Arm FVP (Installed with Arm DS) - VE_Cortex_A9x1
cd "C:\Users\yourname\Development Studio Workspace"
Working directory "C:\Users\yourname\Development Studio Workspace"
Execution stopped in SVC mode at S:0x00000000
S:0x00000000   DCI      0xe7ff0010 ; ? Undefined
source jython_progress/progress_monitor.py
Lorem ipsum dolor sit amet, consectetur adipiscing elit
Duis neque nibh, facilisis sed rutrum vitae, lobortis feugiat dolor
You can cancel any sentence monitor or word sub-monitor without cancelling the whole script, try it by clicking one of the red stop buttons.
In this example, the Jython script progress_monitor.py is executed as one of a set of debugger commands that are executed automatically when a connection is established. It is also possible to invoke the Jython script manually:
In the Command field of the Command view, enter the source command to run progress_monitor.py, for example:
source jython_progress/progress_monitor.py
Alternatively, "drag and drop" the progress_monitor.py file from the Project Explorer view into the Scripts view, then double-click on it to execute it. The Jython Script Parameters control in the Scripts view also allows you to specify parameters to be passed to a script.
