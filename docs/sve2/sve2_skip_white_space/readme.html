SVE2 Skip White Space example - Arm®Development Studio

A simple C application, showing the use of Scalable Vector Extension 2 (SVE2) assembler to perform "skip white space".
Note
For information on locating the example files and extracting them for use either on the command-line or in Arm Development Studio, see the installation instructions.

Purpose and scope

This is a simple C application, that shows the use of Scalable Vector Extension 2 (SVE2) assembler to perform a "skip white space" function.
The "skip white space" function uses the SVE2 NMATCH instruction to skip the leading white space in a string.
This example illustrates how build SVE2 assembler with Arm Compiler for Embedded 6, run it on an FVP model, and debug it with the Debugger.
This example is derived from the "skipwhitespace" example provided with these SVE Programming Examples.

Building the example

A ready-made executable sve2_skip_white_space.axf is provided, pre-built with Arm Compiler for Embedded 6.
If you wish to modify or rebuild this example, you can do so using the supplied Eclipse (makefile builder) project, or directly on the command line with the supplied Makefile.

Building on the command-line

To build the example on the command-line with the supplied make utility:
  • On Windows, open an Arm DS Command Prompt from the Start menu, run the select_toolchain utility, and select Arm Compiler for Embedded 6 from the list
  • On Linux, run the suite_exec utility with the --toolchain option to select the compiler and start a shell configured for the Development Studio environment, for example: ~/developmentstudio/bin/suite_exec --toolchain "Arm Compiler for Embedded 6" bash
Then navigate to the ...\sve2_skip_white_space directory, and type:
make
The usual make rules: clean and all are provided in the makefile.

Building within the IDE

To build the supplied projects within the IDE:
  1. In the Project Explorer view, select the project you want to build.
  2. Select Project → Build Project.

Running the example

  1. Select Run → Debug Configurations....
  2. Select sve2_skip_white_space from the list of Generic Arm C/C++ Application configurations.
  3. Click on Debug to start debugging. The executable image will be downloaded to the target, then starts executing, and stops at main().
  4. Run the executable by clicking on the green Continue button in the Debug Control view, or by pressing F8 on the keyboard. The following output is displayed in the Target Console.
    SVE2 Skip White Space example using NMATCH instruction

Input string  = '   Hello   '
Output string = 'Hello   '
  5. Disconnect the debug connection by double-clicking on sve2_skip_white_space that is now in the Debug Control view.
You can also run the example executable directly on the FVP model from within the Arm DS Command Prompt on Windows or the suite_exec shell on Linux with, for example:
FVP_Base_Neoverse-N2x1 --plugin="[Arm DS installation folder]/sw/models/bin/ScalableVectorExtension.so" -C SVE.ScalableVectorExtension.has_sve2=1 -C bp.secure_memory=false -a sve2_skip_white_space.axf

Debugging the example

  1. Re-establish the debug connection by double-clicking on sve2_skip_white_space in the Debug Control view.
  2. In main.c, observe that main() first initializes a test string, then calls DisableSveTraps(), then calls SkipWhiteSpace(). Both DisableSveTraps and SkipWhiteSpace are functions written in assembler in skip_white_space.S. SkipWhiteSpace takes two parameters - the start and end of the string, in x0 and x1 respectively.
  3. Open skip_white_space.S and set a breakpoint on both DisableSveTraps and SkipWhiteSpace.
  4. In the Registers view, expand AArch64 > System > Secure > CPTLR_EL3. Notice the EZ bit is initially 0 and the TFP bit is set to 1, meaning SVE instructions will be trapped, unless these bits are inverted.
  5. Single-step over the MRS, BIC, ORR, MSR, ISB instructions, and observe the EZ bit being set to 1 and the TFP bit being cleared, thereby disabling the SVE and SIMD traps.
  6. Continue running to the breakpoint on SkipWhiteSpace.
  7. In the Registers view, expand AArch64 > SVE > Data > Z1 > S32. Right-mouse-click on Z1.S32[0] and select display in Hexadecimal.
  8. Single-step over the MOV w3, ADD w3, MOV z1.s instructions, and observe the Z1.S32[0] being set to 0x0D090A20, which is ASCII for '\r\t\n '.
  9. In the Registers view, expand AArch64 > SVE > Predicate.
  10. Single-step over the WHILELT instruction, and observe the predicate register P0 being set to 0x7FF, representing the 11 characters of the input string.
  11. To see a full description of the WHILELT instruction, or any other instruction in this example, simply hover the mouse over the instruction, in either the source editor or Disassembly views.
  12. In the Registers view, expand AArch64 > SVE > Data > Z0 > S8. Right-mouse-click on Z0.S8[0] and select display as a Character.
  13. Single-step over the LD1B instruction, and observe Z0.S8 being set to the charcters of the input string.
  14. Single-step over the NMATCH instruction, and observe the predicate register P1 being set to 0xF8, representing the positions of the 5 non-white-space characters of the input string.
  15. Single-step to and over the BRKB instruction, and observe the predicate register P2 being set to 0x7, representing the 3 leading white-space characters of the input string.
  16. Single-step over the INCP instruction, and observe x0 being incremented by 3, thereby discarding the leading white-space characters.
  17. Single-step over the MOV and RET instructions, which return the start of the resulting string, stripped of its leading white-space characters, in x2.
  18. Continue running the executable by clicking on the green Continue button in the Debug Control view, or by pressing F8 on the keyboard.
To capture a history of instruction execution ("trace") from the FVP model and view it in the Debugger, first disconnect any existing debug session, then:
  1. In the Project Explorer view, double-click on sve2_skip_white_space.launch.
  2. In the Edit Configuration dialog, press the "DTSL Options" Edit... button.
  3. In the DTSL Configuration dialog, select Model Trace in the Trace Buffer tab, and select Enable Neoverse N2 trace in the Core Trace tab, then press OK.
  4. In the Edit Configuration dialog, press the Debug button.
  5. Open the Trace view.
  6. Run and debug as before. A history of instruction execution is displayed in the Trace view.
